Los sistemas de control de versiones

El control de versiones (VCS por sus siglas en ingles), es una herramienta la cual consiste en un sistema que lleva a cabo un registro de los cambios realizados a archivo, o por ejemplo, un proyecto a lo largo de su desarrollo de manera de que se puedan recuperar diferentes versiones del proyecto en el caso que sea necesario ya siendo por que hay un error o por que se perdió el proyecto original.

Clasificaciones

Podemos clasificar los VCS en tres tipos diferentes, cada uno teniendo una característica que logra que se diferencie de las demás.

Sistemas de control de versiones locales

Este siendo local, se basa en una base de datos sencilla en la se llevará acabo el registro de los cambios realizados a los archivos

Sistemas de control de versiones centralizados

Los VCS centralizados almacenan los registros de los cambios hechos dentro de un único servidor manteniéndolos versionados, este sistema hace posible que varios usuarios puedan descargar diferentes versiones directamente desde el servidor, aunque con la desventaja que, si el servidor no esta disponible o su almacenamiento sufre algún percance, todas las versiones almacenadas ya no estarán disponibles además de que no podrán guardar nuevas versiones.

Sistemas de control de versiones distribuidos

Similar a los sistemas de control centralizados, en los distribuidos también se guardan las versiones en un servidor con la diferencia los usuarios no únicamente descargan la versión más reciente, sino que también se replica el repositorio completo de esta manera solucionando el problema de no tener acceso a las versiones en el caso que el servidor este caído. También se pueden utilizar estos repositorios duplicados para restaurar el servidor original.

Conclusión
Los sistemas de control de versiones

Llevar un control de versiones es una buena practica debido a la naturaleza de como se trabaja el desarrollo de aplicaciones de software siendo que con frecuencia se pueden cometer errores de los cuales no se encuentra la solución o perder el trabajo ya realizado es un gran contratiempo.
